## 摘要
无评论员（Critic-free）的带可验证奖励的强化学习（RLVR）通过优化单元测试通过率来改进代码生成，但 GRPO 风格的更新面临粗粒度信用分配的问题：即使失败源于局部的语义错误，单一的结果信号也会均匀地分布在长程序中。我们提出了基于执行的信用分配（EGCA），它利用执行轨迹使 GRPO 更新局部化。对于满足算法约束但未通过测试的程序，EGCA 在相同的插桩环境下执行候选方案和标准参考解（离线整理一次；用于分析而非监督），识别最早的语义分歧点，并仅对相应的 token 跨度分配优势（advantage），同时屏蔽下游 token。EGCA 是一种即插即用的修改，不需要评论员、辅助损失或学习型验证器，在 HumanEval 上实现了 82.1% 的 pass@1（比 GRPO 提高 3.1%），在 MBPP 上实现了 68.9%（提高 1.5%），且仅有 18% 的实际运行时间开销。

## Abstract
Critic-free reinforcement learning with verifiable rewards (RLVR) improves code generation by optimizing unit-test pass rates, but GRPO-style updates suffer from coarse credit assignment: a single outcome signal is spread uniformly across long programs even when failure stems from a localized semantic error. We propose Execution-Grounded Credit Assignment (EGCA), which localizes GRPO updates using execution traces. For programs that satisfy algorithmic constraints but fail tests, EGCA executes the candidate and a canonical reference solution (curated once offline; used for analysis, not supervision) under identical instrumentation, identifies the earliest semantic divergence, and assigns advantage only to the corresponding token span while masking downstream tokens. EGCA is a drop-in modification requiring no critic, auxiliary loss, or learned verifier, yielding 82.1% pass@1 on HumanEval (+3.1 over GRPO) and 68.9% on MBPP (+1.5) with 18% wall-clock overhead.

### 论文总结：代码生成中基于执行的 GRPO 信用分配 (EGCA)

#### 1. 论文的核心问题与整体含义
在代码生成领域，利用可验证奖励（如单元测试）进行强化学习（RLVR）已成为主流。然而，现有的无评论员（Critic-free）算法（如 GRPO）面临**信用分配（Credit Assignment）粗糙**的问题：当一个长程序因为局部的一个小语义错误（如符号写反、边界条件错误）而导致测试失败时，算法会将负面信号均匀地分配给整个程序的所有 token。这种“信用涂抹”现象导致模型难以从接近正确的样本中精准学习。本文认为，在模型已经能生成语法正确代码的阶段，**信用分配精准度而非奖励稀疏性**才是提升性能的关键瓶颈。

#### 2. 方法论 (EGCA)
论文提出了 **EGCA (Execution-Grounded Credit Assignment)**，其核心思想是利用执行轨迹（Execution Traces）将全局奖励局部化到具体的错误代码段。
*   **核心流程**：
    1.  **约束引导采样**：利用离线整理的参考解提取算法约束，引导模型生成结构相似的候选代码。
    2.  **故障模式路由**：将生成的代码按优先级归类为：语法错误 (SYNTAX)、约束违规 (CONSTRAINT)、逻辑错误 (LOGIC) 或完全正确 (CORRECT)。
    3.  **执行分歧定位**：对于“逻辑错误”类代码（即能运行但结果错），在相同输入下同时执行候选代码和参考解，记录状态轨迹。
    4.  **信用分配算子**：识别两者轨迹中**最早出现语义分歧**的步骤，将 GRPO 的优势值（Advantage）集中分配给该步骤对应的 token 跨度，并**屏蔽（Mask）该跨度之后的所有 token**。
*   **公式体现**：在 GRPO 目标函数中，将原本均匀分布的 token 级优势 $a_{i,t}$ 修改为基于分歧跨度 $T_{k^*}$ 的加权分布，仅在关键跨度内保留梯度贡献。

#### 3. 实验设计
*   **数据集**：训练集使用 **APPS+**（包含自然语言描述、单元测试和参考解）；评估集使用 **HumanEval** 和 **MBPP**。
*   **基座模型**：DeepSeek-Coder-Instruct-6.7B。
*   **对比方法**：
    *   Vanilla GRPO（均匀信用分配）。
    *   StepCoder（仅屏蔽未执行的 token）。
    *   RLTF（利用细粒度单元测试反馈）。
    *   CodeRL+（引入辅助执行语义对齐损失）。
*   **调试器模型**：使用 Qwen2.5-Coder-7B-Instruct 作为定位分歧的工具模型。

#### 4. 资源与算力
*   **硬件配置**：使用了 **8 张 NVIDIA A100 80GB GPU**。
*   **训练参数**：训练 3 个 epoch，全局 Batch Size 为 64，学习率为 $5 \times 10^{-7}$。
*   **开销**：相比原生 GRPO，EGCA 引入了执行轨迹分析，带来了约 **18% 的实际运行时间（Wall-clock）额外开销**。

#### 5. 实验数量与充分性
*   **实验规模**：在两个主流榜单上进行了完整测试，并进行了多项消融实验。
*   **消融实验**：对比了定位到“最早分歧点” vs “最后分歧点” vs “随机执行边界”的效果，验证了定位最早分歧点的必要性。
*   **客观性验证**：专门设计了“教师泄漏”控制实验。结果显示，即使使用性能较弱的 1.5B 模型作为调试器，训练出的 6.7B 学生模型性能仍远超调试器本身（高出 8.2%），证明了提升源于信用分配的优化而非简单的知识蒸馏。

#### 6. 主要结论与发现
*   **性能提升**：EGCA 在 HumanEval 上达到 82.1% (+3.1% vs GRPO)，在 MBPP 上达到 68.9% (+1.5% vs GRPO)。
*   **定位价值**：在接近正确的代码中，知道“哪里开始出错”比单纯知道“出错了”更有价值。
*   **无需评论员**：在不引入复杂的 Critic 网络或辅助损失函数的情况下，仅通过修改梯度分配权重即可显著提升 RL 效果。

#### 7. 优点
*   **即插即用**：作为 GRPO 的改进版，不需要额外的价值网络（Value Network），降低了显存占用和架构复杂度。
*   **语义驱动**：相比于 StepCoder 仅关注代码是否被执行，EGCA 深入到了执行状态的语义层面。
*   **鲁棒性**：实验证明该方法对调试器模型的水平要求不高，具有较强的普适性。

#### 8. 不足与局限
*   **依赖参考解**：需要预先存在正确的参考代码来对比轨迹，这限制了其在完全开放式或无参考解场景下的应用。
*   **阶段依赖性**：该方法在模型已经具备一定基础能力（即进入“接近正确”阶段）时最有效；如果模型生成的代码大多无法运行，该方法的优势将无法体现。
