## 摘要
提升语言模型数学推理能力的主流范式依赖于具有可验证奖励的强化学习。然而，现有方法孤立地处理每个问题实例，未能利用训练过程中产生并积累的可复用策略。为此，我们提出了 ARISE（通过内在技能演化的智能体推理），这是一个分层强化学习框架，其中共享策略既在顶层管理技能，又在底层生成响应（分别称为技能管理器和执行器）。管理器通过专门的技能生成展开（rollout）来维护分层技能库，该过程对成功的解题轨迹进行结构化总结（执行后），同时采用策略驱动的选择机制来检索相关技能，以指导未来的展开（执行前）。分层奖励设计引导了推理能力与技能库质量的协同演化。在两个基础模型和涵盖竞赛数学及 Omni-MATH 的七个基准测试上的实验表明，ARISE 一贯优于 GRPO 系列算法和记忆增强基线，在分布外任务上的提升尤为显著。消融研究证实，每个组件都对观察到的改进有所贡献，且技能库质量和推理性能在整个训练过程中同步提升。代码可在 https://github.com/Skylanding/ARISE 获取。

## Abstract
The dominant paradigm for improving mathematical reasoning in language models relies on Reinforcement Learning with verifiable rewards. Yet existing methods treat each problem instance in isolation without leveraging the reusable strategies that emerge and accumulate during training. To this end, we introduce ARISE (Agent Reasoning via Intrinsic Skill Evolution), a hierarchical reinforcement learning framework, in which a shared policy operates both to manage skills at high-level and to generate responses at low-level (denoted as a Skills Manager and a Worker, respectively). The Manager maintains a tiered skill library through a dedicated skill generation rollout that performs structured summarization of successful solution traces (after execution), while employing a policy-driven selection mechanism to retrieve relevant skills to condition future rollouts (before execution). A hierarchical reward design guides the co-evolution of reasoning ability and library quality. Experiments on two base models and seven benchmarks spanning both competition mathematics and Omni-MATH show that ARISE consistently outperforms GRPO-family algorithms and memory-augmented baselines, with particularly notable gains on out-of-distribution tasks. Ablation studies confirm that each component contributes to the observed improvements and that library quality and reasoning performance improve in tandem throughout training. Code is available at \href{https://github.com/Skylanding/ARISE}{https://github.com/Skylanding/ARISE}.

#### 1. 核心问题与整体含义（研究动机和背景）
当前提升大语言模型（LLM）数学推理能力的主流方法（如 DeepSeek-R1 采用的 GRPO）通常将每个问题实例孤立对待。模型在训练过程中产生的成功解题策略（如特定的代数技巧或几何辅助线思路）在单次更新后即被丢弃，无法积累并复用。
**研究动机：** 为智能体装备一个持久的、可进化的“技能库”，使其能够跨问题迁移知识，避免冗余探索，并解决现有方法中技能管理与策略优化脱节的问题。

#### 2. 方法论：核心思想与关键技术
ARISE 提出了一个**分层强化学习（HRL）框架**，其核心是将单一策略 $\pi_\theta$ 同时作为“技能管理器（Manager）”和“执行者（Worker）”：
*   **核心思想：** 技能库不再是外部静态资源，而是智能体状态的内在组成部分，随策略同步演化。
*   **技能选择（执行前）：** 管理器通过计算查询（Query）与技能文档之间的**条件对数概率（Log-probability）**进行评分，采用 $\epsilon$-greedy 策略和置信度门控机制选择最相关的技能注入 Prompt。
*   **技能生成与维护（执行后）：** 在标准的 $G$ 组解题展开之外，增加一个专门的 $O_{G+1}$ 展开，对成功的解题轨迹进行结构化总结，生成 JSON 格式的技能文档。
*   **两层库架构：** 采用“缓存（Cache）- 仓库（Reservoir）”架构，通过 `ADD`（添加）、`UPDATE`（更新效用）、`EVICT`（驱逐）、`LOAD`（加载）和 `DELETE`（删除）五种操作动态维护技能质量。
*   **分层奖励设计：** 引入三级奖励 $R \in \{0, 1, 2\}$：
    *   $r_0=0$：解答错误。
    *   $r_1=1$：解答正确但未使用技能。
    *   $r_2=2$：解答正确且成功应用技能。
    这种设计通过组相对优势（Group-relative advantage）引导模型优先学习如何有效利用技能。

#### 3. 实验设计
*   **基础模型：** Qwen3-4B-Instruct-2507 和 Phi-4-mini-instruct (3.8B)。
*   **训练数据：** DeepScaleR 数据集（约 4 万个问题，涵盖 AMC、AIME、MATH 等）。
*   **基准测试（7 个）：** 
    *   分布内（In-distribution）：AMC 2023, AIME 2024, AIME 2025。
    *   分布外（OOD）：Omni-MATH 的四个领域（代数、数论、组合、几何）。
*   **对比方法：** 
    *   **GRPO 家族：** 原生 GRPO, Dr.GRPO, DAPO, GSPO。
    *   **记忆/技能增强类：** EvolveR, SimpleMem, SkillRL。

#### 4. 资源与算力
*   **硬件：** 使用了 **8 张 NVIDIA A100-80GB GPU**。
*   **训练细节：** 训练时长为 2000 个优化步（Steps）。
*   **参数设置：** 组大小 $G=8$，有效 Batch Size 为 64，最大生成长度 4096 标记。
*   **开销：** 相比原生 GRPO，ARISE 增加了约 16.6% 的训练耗时（主要来自技能评分和 $O_{G+1}$ 展开）。

#### 5. 实验数量与充分性
*   **实验规模：** 针对两个不同架构的基础模型进行了完整训练和评估。
*   **评估稳健性：** 所有 Pass@1 准确率均基于 **32 次独立运行的平均值**，显著降低了随机性干扰。
*   **消融实验：** 针对分层奖励、随机选择、技能生成机制、置信度门控等关键组件进行了详尽的消融分析。
*   **客观性：** 实验对比了当前最先进的多种 GRPO 变体，且在分布外任务（Omni-MATH）上进行了深度测试，验证了泛化能力。

#### 6. 主要结论与发现
*   **性能领先：** ARISE 在所有基准测试上均优于 GRPO 系列和现有的记忆增强方法。
*   **泛化优势：** 在 OOD 任务（Omni-MATH）上提升尤为显著（Qwen3 提升 2.9%），表明技能库有助于处理未见过的复杂数学领域。
*   **协同演化：** 实验观察到技能库的质量（效用值）与模型的推理准确率同步增长，且后期性能提升更多源于“更精准的技能选择”而非“库规模的扩大”。
*   **效率平衡：** 尽管引入了技能注入，但通过严格限制技能文档长度（220 字符），ARISE 在推理开销与性能增益之间达到了最优平衡。

#### 7. 优点
*   **内生性优化：** 技能选择直接受策略梯度驱动，而非依赖外部不可微的检索器，实现了端到端优化。
*   **结构化知识积累：** 将零散的解题经验转化为结构化的 JSON 技能，增强了模型的可解释性。
*   **分层奖励机制：** 巧妙利用组相对优势解决了“技能使用动机”问题，强制模型在正确解题的基础上追求更优的策略应用。

#### 8. 不足与局限
*   **领域局限：** 目前仅在数学推理领域验证，尚未扩展到代码生成或多工具调用等更通用的智能体任务。
*   **计算开销：** 训练时每个 Step 需要额外的 $O_{G+1}$ 展开和多次前向传播评分，对算力有一定额外要求。
*   **库饱和风险：** 随着训练进行，技能库可能会出现冗余或饱和，如何进行更高级的技能抽象（而非简单的总结）仍是挑战。
