## 摘要
Text-to-SQL 解析在“全模式假设”下取得了显著进展。然而，这一前提在现实世界的企业环境中往往失效，因为那里的数据库包含数百张表以及海量的噪声元数据。智能体必须主动识别并验证相关的子集，而不是预先注入完整的模式，这便产生了我们在本研究中探讨的“未知模式”场景。为此，我们提出了 TRUST-SQL（通过工具在未知模式下进行真实推理）。我们将该任务建模为一个部分可观测马尔可夫决策过程（POMDP），其中我们的自主智能体采用结构化的四阶段协议，将推理建立在经过验证的元数据之上。至关重要的是，该协议为我们新颖的双轨 GRPO（Dual-Track GRPO）策略提供了结构边界。通过应用 Token 级的掩码优势，该策略将探索奖励与执行结果隔离开来，以解决信用分配问题，相比标准 GRPO 实现了 9.9% 的相对提升。在五个基准测试上的广泛实验表明，TRUST-SQL 的 4B 和 8B 变体相较于其基础模型，平均绝对提升分别达到了 30.6% 和 16.6%。值得注意的是，尽管在完全没有预加载元数据的情况下运行，我们的框架仍能持续匹配或超越依赖模式预填充的强基准模型。

## Abstract
Text-to-SQL parsing has achieved remarkable progress under the Full Schema Assumption. However, this premise fails in real-world enterprise environments where databases contain hundreds of tables with massive noisy metadata. Rather than injecting the full schema upfront, an agent must actively identify and verify only the relevant subset, giving rise to the Unknown Schema scenario we study in this work. To address this, we propose TRUST-SQL (Truthful Reasoning with Unknown Schema via Tools). We formulate the task as a Partially Observable Markov Decision Process where our autonomous agent employs a structured four-phase protocol to ground reasoning in verified metadata. Crucially, this protocol provides a structural boundary for our novel Dual-Track GRPO strategy. By applying token-level masked advantages, this strategy isolates exploration rewards from execution outcomes to resolve credit assignment, yielding a 9.9% relative improvement over standard GRPO. Extensive experiments across five benchmarks demonstrate that TRUST-SQL achieves an average absolute improvement of 30.6% and 16.6% for the 4B and 8B variants respectively over their base models. Remarkably, despite operating entirely without pre-loaded metadata, our framework consistently matches or surpasses strong baselines that rely on schema prefilling.

### 1. 核心问题与研究动机
*   **核心问题**：传统的 Text-to-SQL 研究大多基于“全模式假设”（Full Schema Assumption），即假设数据库的所有表结构和元数据都能预先加载到模型的上下文窗口中。
*   **研究动机**：在真实企业环境中，数据库往往包含数百张表和海量噪声元数据，超出了上下文限制且易导致模型幻觉。论文提出了**“未知模式”（Unknown Schema）**场景，要求智能体像人类专家一样，通过主动探索、验证元数据来构建 SQL，而非被动接收信息。

### 2. 方法论
*   **核心思想**：将 Text-to-SQL 建模为**部分可观测马尔可夫决策过程（POMDP）**，通过结构化的交互协议和改进的强化学习算法，使模型具备自主探索和自我纠错能力。
*   **关键技术细节**：
    *   **四阶段交互协议（EPGC）**：
        1.  **Explore（探索）**：调用工具查询数据库元数据。
        2.  **Propose（提议）**：强制性的认知检查点，要求模型提交已验证的模式子集，有效抑制幻觉。
        3.  **Generate（生成）**：基于提议的模式生成候选 SQL。
        4.  **Confirm（确认）**：执行并提交最终结果。
    *   **双轨 GRPO（Dual-Track GRPO）**：
        *   在 DeepSeek 的 GRPO 基础上，将轨迹划分为**模式轨道（Schema Track）**和**全量轨道（Full Track）**。
        *   **令牌级掩码优势（Token-level Masked Advantages）**：通过掩码技术，确保探索奖励仅影响探索阶段的 Token，执行奖励影响全量 Token，解决了长路径下的信用分配（Credit Assignment）难题。
    *   **奖励设计**：包含执行奖励（$R_{exec}$）、格式奖励（$R_{fmt}$）和模式匹配奖励（$R_{schema}$）。

### 6. 主要结论与发现
*   **性能飞跃**：TRUST-SQL 使 4B 和 8B 模型在未知模式下的平均绝对准确率分别提升了 **30.6%** 和 **16.6%**。
*   **无需预填充**：即使不预加载任何模式信息，TRUST-SQL 的表现也优于许多依赖模式注入的强基准模型。

*   **测试时缩放**：增加推理时的交互轮数和采样次数（Pass@K）能显著持续提升性能。

### 7. 优点（亮点）
*   **现实意义强**：打破了学术界常用的“全模式假设”，更贴近工业界真实的数据库交互场景。
*   **算法创新**：Dual-Track GRPO 巧妙地利用协议边界解决了多轮对话中的信用分配问题，为 Agent 类任务的强化学习提供了新思路。
*   **高效性**：相比于 CHESS 等依赖长上下文的框架，TRUST-SQL 在推理延迟和 Token 消耗上实现了百倍级的降低。

### 8. 不足与局限
*   **推理开销**：多轮交互模式虽然准确，但相比单轮生成，其推理延迟和数据库调用次数更高。
*   **方言限制**：目前主要针对 **SQLite** 进行优化和评估，尚未广泛验证在 PostgreSQL 或 MySQL 等其他 SQL 方言上的表现。
*   **固定轮数预算**：训练时设定了固定的最大轮数（如 10 轮），对于极度复杂的数据库模式，可能需要更动态的轮数分配机制。
