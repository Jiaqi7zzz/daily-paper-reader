## 摘要
当答案依赖于即时信息或冲突信息时，大语言模型（LLMs）在事实性问答任务中仍然表现得不够可靠。尽管检索增强和使用工具的 LLMs 减少了幻觉，但它们通常依赖隐式规划，导致工具使用效率低下。我们提出了一个模块化框架，将规划与事实检索及答案合成明确分离。通过教师-学生框架训练一个轻量级学生规划器，以生成由抽象推理步骤和可搜索事实请求组成的结构化分解。监督信号仅包含规划轨迹和事实请求，不提供事实性答案或检索到的证据。在推理阶段，规划器生成计划，而经过提示工程优化的模块则执行检索和响应合成。我们在 SEAL-0 上评估了所提框架，这是一个针对搜索增强型 LLMs 的极具挑战性的基准测试。结果表明，与单体推理模型和基于提示的工具增强框架相比，有监督规划提高了准确性并降低了延迟，证明了显式学习的规划结构对于构建可靠的事实性 LLMs 至关重要。

## Abstract
Fact-seeking question answering with large language models (LLMs) remains unreliable when answers depend on up-to-date or conflicting information. Although retrieval-augmented and tool-using LLMs reduce hallucinations, they often rely on implicit planning, leading to inefficient tool usage. We propose a modular framework that explicitly separates planning from factual retrieval and answer synthesis. A lightweight student planner is trained via a teacher-student framework to generate structured decompositions consisting of abstract reasoning steps and searchable fact requests. The supervision signals contain only planning traces and fact requests, without providing factual answers or retrieved evidence. At inference, the planner produces plans, while prompt-engineered modules perform retrieval and response synthesis. We evaluate the proposed framework on SEAL-0, an extremely challenging benchmark for search-augmented LLMs. Results show that supervised planning improves both accuracy and latency compared to monolithic reasoning models and prompt-based tool-augmented frameworks, demonstrating that explicitly learned planning structures are essential for reliable fact-seeking LLMs.

*   **核心理念**：提出将**推理逻辑（Reasoning）**与**事实知识（Knowledge）**彻底解耦。通过训练一个只负责“规划”而不存储“事实”的轻量级模型，确保所有事实性陈述都必须源自外部可验证的证据。

### 2. 方法论：核心思想与关键技术
该框架将推理过程模块化为四个阶段：**规划（Planning）、检索（Retrieval）、事实提取（Extraction）和答案聚合（Aggregation）**。
*   **教师-学生蒸馏框架**：
    *   **教师模型**：使用 GPT-5.2 作为教师，为 1,596 个复杂问题生成结构化计划（JSON 格式）。计划包含：1) 抽象推理步骤；2) 原子级事实请求（如 Web 搜索或计算指令）。教师被严禁直接给出答案。
    *   **学生规划器（核心）**：对 Qwen 3-8B 进行微调。**关键创新点**在于监督信号中**不包含任何事实答案或检索证据**，只包含规划轨迹。这使得学生模型学会的是“如何寻找信息”的逻辑，而非“记住信息”。
*   **推理流程**：
    1.  **学生规划器**生成 JSON 计划。
    2.  **检索模块**通过 SerpAPI（Google Search）执行搜索。
    3.  **提取模块**（提示词工程化的 LLM）从原始搜索结果中提取简洁事实。
    4.  **计算模块**处理数值比较、日期计算等逻辑。
    5.  **聚合模块**根据检索到的事实和规划步骤合成最终答案。

*   **数据集/基准测试**：主要使用 **SEAL-0**（SEAL-QA 的核心子集），包含 111 个极具挑战性的事实搜索问题。这些问题的特点是：即使是带有联网功能的强力模型（如 GPT-4o）也经常得到 0% 的准确率。

*   **消融/对比实验**：对比了“有无搜索”、“有无微调规划器”、“有无内部推理”等多种配置，实验设计较为客观。

### 6. 主要结论与发现
*   **准确性提升**：在 SEAL-0 上，本框架（学生规划器）达到了 **10.8%** 的准确率，显著优于单体模型的 1.8% 和提示词框架的 6.3%。这一成绩在同类开源模型中处于领先地位。

### 7. 优点
*   **解耦彻底**：通过“不含知识的蒸馏”，有效解决了模型因记忆错误知识而产生的幻觉。
*   **轻量高效**：使用 8B 规模的模型即可在复杂任务上挑战甚至超越闭源大模型。
*   **可解释性强**：模块化设计使得错误可以追溯到具体的规划、检索或提取阶段。
*   **鲁棒性**：学生规划器生成的 JSON 格式更稳定，减少了因格式错误导致的推理失败。

### 8. 不足与局限
*   **检索依赖**：框架性能高度依赖外部搜索引擎（SerpAPI）的质量。如果搜索结果本身有误或缺失，模型无法纠正。
*   **延迟瓶颈**：尽管比基准快，但受限于多次外部 API 调用和多步推理，实时性仍有待提升。

*   **计算模块限制**：目前的计算模块依赖提示词工程，对于极复杂的数学或逻辑运算可能仍存在瓶颈。
