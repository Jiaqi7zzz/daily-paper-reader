## 摘要
提高数据利用效率对于在生成轨迹成本高昂的长程任务中扩展强化学习（RL）至关重要。然而，目前主流的大语言模型（LLM）强化学习方法大多是同策（on-policy）的：它们对每批数据仅更新一次后便将其丢弃，随后采集新样本，导致样本效率低下。在这项工作中，我们探索了一种替代的基于价值的 LLM 强化学习框架，该框架天然支持离策（off-policy）学习。我们提出了 ReVal，这是一种基于贝尔曼更新（Bellman-update）的方法，它结合了捕捉内部一致性的逐步信号与源自结果验证的轨迹级信号。ReVal 天然支持基于经验回放池（replay-buffer）的训练，从而能够高效地复用过去的轨迹。在标准数学推理基准测试上的实验表明，ReVal 不仅收敛更快，而且在最终性能上优于 GRPO。在 DeepSeek-R1-Distill-1.5B 模型上，ReVal 提高了训练效率，相比 GRPO 在 AIME24 上提升了 2.7%，在域外基准测试 GPQA 上提升了 4.5%。这些结果表明，基于价值的强化学习是 LLM 训练中替代基于策略（policy-based）方法的一种实用方案。

## Abstract
Improving data utilization efficiency is critical for scaling reinforcement learning (RL) for long-horizon tasks where generating trajectories is expensive. However, the dominant RL methods for LLMs are largely on-policy: they update each batch of data only once, discard it, and then collect fresh samples, resulting in poor sample efficiency. In this work, we explore an alternative value-based RL framework for LLMs that naturally enables off-policy learning. We propose ReVal, a Bellman-update-based method that combines stepwise signals capturing internal consistency with trajectory-level signals derived from outcome verification. ReVal naturally supports replay-buffer-based training, allowing efficient reuse of past trajectories. Experiments on standard mathematical reasoning benchmarks show that ReVal not only converges faster but also outperforms GRPO in final performance. On DeepSeek-R1-Distill-1.5B, ReVal improves training efficiency and achieves improvement of 2.7% in AIME24 and 4.5% in out-of-domain benchmark GPQA over GRPO. These results suggest that value-based RL is a practical alternative to policy-based methods for LLM training.

*   **核心问题**：当前大语言模型（LLM）的主流强化学习（RL）方法（如 PPO、GRPO、ReMax）大多是**同策（On-policy）**的。这意味着每批采集的数据在更新一次参数后即被丢弃，无法重复利用。
*   **研究背景**：在长程推理任务中，生成完整轨迹的计算成本极高。同策方法极低的样本利用率成为了扩展 LLM 推理能力的瓶颈。
*   **整体含义**：本文旨在探索一种**基于价值（Value-based）的离策（Off-policy）**强化学习框架，通过引入经验回放机制，让模型能够从历史轨迹中多次学习，从而在降低训练成本的同时提升模型性能。

### 2. 方法论：ReVal 框架
*   **核心思想**：将 LLM 的 Logits 直接解释为 Q 值（无需额外的价值网络），并利用贝尔曼更新（Bellman Update）进行离策训练。
*   **关键技术细节**：
    *   **Logit-as-Q 参数化**：利用预训练模型已有的 Logits 作为隐式 Q 函数，避免了引入额外价值模型带来的内存和计算开销。
    *   **奖励塑造与贝尔曼算子**：设计了一种结合了“任务奖励”（结果验证）和“内生奖励”（参考模型一致性）的损失函数。
    *   **校准初始化（Calibrated Initialization）**：针对前人工作（如 TBRM）在无奖励时会产生策略漂移的问题，ReVal 通过数学证明和目标函数设计，确保在奖励为 0 时策略保持在参考模型状态。
    *   **经验回放池（Replay Buffer）**：引入 FIFO（先进先出）缓冲区存储历史轨迹，每轮迭代从池中采样进行多次离策更新。
*   **算法流程**：采样轨迹 -> 存入 Buffer -> 从 Buffer 采样离策数据 -> 计算 ReVal 损失（基于贝尔曼残差） -> 更新参数 -> 定期重置参考策略以缓解 KL 约束。

*   **高效复用**：打破了 LLM 强化学习必须“即用即弃”的传统，显著提升了数据利用率。
