## 摘要
我们提出了一种新的言语强化学习（VRL）框架，用于在存在执行不确定性的移动机器人系统中进行可解释的任务级规划。该框架采用闭环架构，通过与物理环境的交互实现策略的迭代改进。在我们的框架中，大语言模型（LLM）执行器利用视觉语言模型（VLM）评论器生成的结构化自然语言反馈，对可执行的行为树进行反复优化；该评论器负责观察物理机器人及其执行轨迹。与传统的强化学习不同，VRL 中的策略更新直接发生在符号规划层面，无需基于梯度的优化。这实现了透明的推理、显式的因果反馈以及人类可解释的策略演化。我们在真实移动机器人上验证了所提框架，该机器人需在执行不确定性下完成多阶段操作与导航任务。实验结果表明，该框架支持可解释的策略改进、对执行失败的闭环自适应，以及在物理机器人系统上的可靠部署。

## Abstract
We propose a new Verbal Reinforcement Learning (VRL) framework for interpretable task-level planning in mobile robotic systems operating under execution uncertainty. The framework follows a closed-loop architecture that enables iterative policy improvement through interaction with the physical environment. In our framework, executable Behavior Trees are repeatedly refined by a Large Language Model actor using structured natural-language feedback produced by a Vision-Language Model critic that observes the physical robot and execution traces. Unlike conventional reinforcement learning, policy updates in VRL occur directly at the symbolic planning level, without gradient-based optimization. This enables transparent reasoning, explicit causal feedback, and human-interpretable policy evolution. We validate the proposed framework on a real mobile robot performing a multi-stage manipulation and navigation task under execution uncertainty. Experimental results show that the framework supports explainable policy improvements, closed-loop adaptation to execution failures, and reliable deployment on physical robotic systems.

*   **核心思想**：将策略表示为可执行的**行为树（Behavior Trees, BTs）**，利用 LLM 作为“执行者（Actor）”修改 BT 结构，利用视觉语言模型（VLM）作为“评论者（Critic）”分析执行过程并提供反馈。
*   **关键技术细节**：
    *   **策略表示**：使用 BT 保证了模块化和透明度。Actor 直接在预定义的节点库上操作，修改 BT 的逻辑结构或参数。
    *   **LLM Actor**：接收任务定义、环境规范、节点库说明及 Critic 的反馈，生成新的 BT 代码。
    *   **VLM Critic**：通过视觉快照和 BT 执行轨迹进行观察。它在初始、中间、最终三个阶段运行，输出结构化自然语言评价、警报分数（Alarm Score）和置信度。
*   **算法流程**：
    1.  初始化一个简单的 BT。
    2.  在物理机器人上执行 BT。
    3.  VLM Critic 观察执行过程，识别错误（如木块放错位置、方向不对）。
    4.  Critic 生成结构化反馈（文本 + 评分）。
    5.  Actor 根据反馈和实际得分（RealScore）更新 BT。
    6.  重复上述过程直至策略收敛或达到目标。

### 6. 主要结论与发现
*   **反馈质量胜过模型规模**：微调后的 3B 模型表现优于未微调的 7B 模型，说明领域相关的结构化反馈比通用视觉能力更重要。
*   **数值奖励不足**：仅靠数值得分（无 Critic 组）几乎无法实现策略改进，因为 Actor 无法区分是“规划错误”还是“硬件执行随机误差”。
*   **共进化现象**：Actor 会自适应 Critic 的局限性。当 Critic 无法准确识别颜色时，Actor 会倾向于采取更保守的策略（如取消旋转动作以节省时间）。
*   **感知是瓶颈**：引入符号化感知信息（BlockInfo）显著提升了学习速度和稳定性，表明 VRL 的主要瓶颈在于底层感知的可靠性而非高层逻辑推理。

### 7. 优点
*   **高可解释性**：策略更新发生在符号（BT）层面，人类可以直接阅读并理解机器人为何修改其行为逻辑。
*   **无需模拟器**：直接在物理机器人上通过少量尝试实现闭环学习，避开了“模拟到现实（Sim-to-Real）”的鸿沟。
*   **安全性**：引入了“人工报警”机制，当 Critic 发现严重错误或置信度过低时会请求人类干预。

### 8. 不足与局限
*   **感知依赖性**：系统高度依赖 VLM 的观察精度。如果 Critic 误判了环境状态，Actor 可能会做出错误的策略调整。
*   **长时程归因困难**：对于具有延迟影响的错误（例如早期的一个小失误导致后期任务失败），目前的框架在因果溯源上仍有挑战。
*   **任务复杂度限制**：实验主要集中在相对结构化的物流任务中，对于高度动态或非结构化环境的泛化能力尚待验证。
