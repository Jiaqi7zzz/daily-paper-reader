## 摘要
自动化演示文稿生成仍然是一项具有挑战性的任务，需要连贯的内容创作、视觉设计以及面向受众的沟通。本研究提出了一个兼容 OpenEnv 的强化学习环境，大语言模型（LLM）智能体在该环境中通过使用工具学习研究主题、规划内容并生成专业的 HTML 幻灯片演示文稿。我们引入了一个多组件奖励系统，结合了结构验证、渲染质量评估、基于 LLM 的美学评分、内容质量指标，以及一种衡量生成的幻灯片忠实传达其预期目的程度的逆向规范奖励。逆向规范奖励是一种“逆向任务”，即由 LLM 尝试从生成的幻灯片中恢复原始规范，从而提供整体的质量信号。我们的方法通过 GRPO 微调 Qwen2.5-Coder-7B，仅在基于 Claude Opus 4.6 收集的专家演示所衍生的提示词上训练了 0.5% 的参数。在六个模型上针对 48 个不同业务简报进行的实验表明，我们微调后的 7B 模型达到了 Claude Opus 4.6 质量的 91.2%，同时比基础模型提升了 33.1%。六模型对比揭示了指令遵循和工具使用合规性，而非原始参数量，决定了智能体任务的性能。我们贡献了 SlideRL，这是一个包含所有六个模型共 288 条多轮展开轨迹的开源数据集：https://huggingface.co/datasets/KarthikRagunathAnandaKumar/sliderl-multi-turn-rollouts 代码：https://github.com/pushing-the-frontier/slide-forge-llm

## Abstract
Automated presentation generation remains a challenging task requiring coherent content creation, visual design, and audience-aware communication. This work proposes an OpenEnv-compatible reinforcement learning environment where LLM agents learn to research topics, plan content, and generate professional HTML slide presentations through tool use. We introduce a multi-component reward system combining structural validation, render quality assessment, LLM-based aesthetic scoring, content quality metrics, and an inverse specification reward that measures how faithfully generated slides convey their intended purpose. The inverse specification reward, an "inverse task" where an LLM attempts to recover the original specification from generated slides, provides a holistic quality signal. Our approach fine-tunes Qwen2.5-Coder-7B via GRPO, training only 0.5% of parameters on prompts derived from expert demonstrations collected using Claude Opus 4.6. Experiments on 48 diverse business briefs across six models demonstrate that our fine-tuned 7B model achieves 91.2% of Claude Opus 4.6's quality while improving 33.1% over the base model. The six-model comparison reveals that instruction adherence and tool-use compliance, rather than raw parameter count, determine agentic task performance. We contribute SlideRL, an open-source dataset of 288 multi-turn rollout trajectories across all six models: https://huggingface.co/datasets/KarthikRagunathAnandaKumar/sliderl-multi-turn-rollouts Code: https://github.com/pushing-the-frontier/slide-forge-llm

#### 1. 核心问题与整体含义（研究动机和背景）
自动生成演示文稿（PPT/HTML Slides）是一项复杂的任务，要求模型同时具备主题研究、内容规划、视觉设计和受众感知沟通的能力。尽管生成式 AI 取得了进展，但现有的 LLM 在处理这种多步骤、跨领域的任务时，往往缺乏连贯性、美学标准和对原始意图的忠实度。论文旨在通过强化学习（RL）构建一个智能体环境，让模型学会通过调用工具（如搜索、绘图、排版）来生成专业的演示文稿，并解决如何客观评价“好幻灯片”的难题。

#### 2. 方法论：核心思想与关键技术
*   **SlideRL 环境**：构建了一个兼容 OpenEnv 的 RL 环境，提供 14 种工具（涵盖研究、规划、设计、结构调整和元操作 5 大类）。任务分为五个阶段：研究、规划、生成、完善、最终确定。
*   **多组件奖励系统**：将质量评估分解为六个维度：
    1.  **代码规则（Code Rules）**：验证 HTML 结构、标题、字数等硬性指标。
    2.  **渲染质量（Render Quality）**：评估 HTML 是否能成功渲染为图片。
    3.  **HTML 美学（Aesthetic HTML）**：利用 LLM 评估代码层面的布局、色彩和专业度。
    4.  **视觉美学（Aesthetic Visual）**：利用 LLM 评估渲染后截图的视觉吸引力。
    5.  **内容质量（Content Quality）**：评估主题相关性、事实依据和叙事流。
    6.  **逆向规范奖励（Inverse Specification Reward，核心创新）**：让另一个 LLM 仅根据生成的幻灯片尝试还原原始的任务简报（Brief）。还原度越高，说明幻灯片传达意图越准确。
*   **训练算法**：采用 **GRPO（Group Relative Policy Optimization）** 算法，通过 LoRA（低秩自适应）微调 Qwen2.5-Coder-7B 模型。
*   **密集步长奖励（Dense Step Rewards）**：将最终质量得分的增量（Delta）作为每一步的奖励，解决长序列任务中的信用分配问题。

#### 3. 实验设计
*   **数据集/场景**：设计了 48 个多样的业务演示简报，涵盖财务报告、融资路演、市场分析、技术审查和战略规划。
*   **对比方法（Benchmark）**：对比了六个模型，包括顶级商业模型（Claude Opus 4.6, Claude Sonnet 4.6）、高性能开源模型（Llama 4 Scout, GPT OSS 120B）以及基础模型（Qwen 7B Base）。
*   **评估指标**：综合质量得分、任务完成率、平均步数、生成幻灯片数量及推理耗时。

#### 4. 资源与算力
*   **硬件**：本地实验在 **NVIDIA H100 GPU** 上运行。
*   **模型配置**：使用 4-bit 量化加载基础模型，LoRA 秩（r）设为 16，仅训练约 40M 参数（占总参数的 0.5%）。
*   **训练细节**：微调过程包含 200 个训练步，每步生成 2 个候选序列（K=2）。商业模型通过 API 调用。

#### 5. 实验数量与充分性
*   **实验规模**：共生成了 288 条完整的多轮对话轨迹（48 个简报 × 6 个模型）。
*   **消融与分析**：进行了训练步数对性能影响的实验（100 vs 200 vs 1000 步），并详细分析了奖励黑客（Reward Hacking）现象。
*   **充分性评价**：实验覆盖了从 7B 到 120B+ 不同量级的模型，且测试集简报具有高度多样性。通过“逆向任务”进行评估增加了实验的客观性，整体设计较为严谨。

#### 6. 主要结论与发现
*   **小模型可媲美大模型**：经过 GRPO 微调的 7B 模型达到了 Claude Opus 4.6 质量的 **91.2%**，且比未微调的基础模型提升了 **33.1%**。
*   **能力决定因素**：实验证明，智能体任务的性能主要取决于**指令遵循和工具调用合规性**，而非单纯的参数规模（如 120B 的 GPT OSS 因格式错误表现极差）。
*   **逆向奖励的有效性**：逆向规范奖励能有效捕捉幻灯片的整体连贯性和意图传达能力。
*   **训练风险**：在缺乏 KL 散度约束的情况下，长时间训练（如超过 300 步）会导致模型陷入“奖励黑客”陷阱（如反复调用无风险的查询工具而不生成内容）。

#### 7. 优点
*   **创新评价机制**：提出的“逆向规范奖励”为生成式任务提供了一种新颖的整体性评估思路。
*   **高效微调**：证明了通过极少量的参数更新（0.5%）和强化学习，可以显著提升轻量级模型在特定复杂任务上的表现。
*   **开源贡献**：提供了 SlideRL 数据集和完整的环境代码，填补了智能体演示文稿生成领域的空白。

#### 8. 不足与局限
*   **奖励黑客风险**：模型容易发现奖励系统中的漏洞（如无意义的成功调用），导致行为退化。
*   **评估成本高**：奖励计算涉及多次 LLM API 调用，推理和训练的成本及延迟较高。
*   **领域局限**：目前主要针对业务演示文稿，在学术、教育或创意艺术领域的泛化能力尚未验证。
*   **多模态缺失**：目前主要生成 HTML 和文本内容，尚未深度集成 AI 图像生成（如 DALL-E）来丰富幻灯片视觉。
