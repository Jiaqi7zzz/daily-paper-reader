## 摘要
用于代码生成的强化学习依赖于来自单元测试通过率的可验证奖励。然而，高质量的测试套件非常稀缺，现有数据集的覆盖范围有限，且静态奖励无法随着模型的改进而自适应。最近的自博弈（self-play）方法将代码生成和测试生成统一在单个模型中，但面临一个固有的困境：白盒访问会导致“自我勾结”（self-collusion），即模型为了轻松获得奖励而生成琐碎的测试；而黑盒限制则会产生通用的测试，从而遗漏特定于实现的错误。我们引入了 Code-A1，这是一个对抗性协同演化框架，它共同优化了具有相反目标的 Code LLM 和 Test LLM。Code LLM 因通过更多测试而获得奖励，而 Test LLM 则因暴露更多缺陷而获得奖励。这种架构上的分离消除了自我勾结的风险，并安全地实现了白盒测试生成，其中 Test LLM 可以检查候选代码以构建有针对性的对抗性测试。我们进一步引入了用于经验回放的“错题本”（Mistake Book）机制，以及一种平衡测试有效性与对抗难度的复合奖励。在 Qwen2.5-Coder 模型上的实验表明，Code-A1 实现了与在人工标注测试上训练的模型相当或更优的代码生成性能，同时显著提升了测试生成能力。

## Abstract
Reinforcement learning for code generation relies on verifiable rewards from unit test pass rates. Yet high-quality test suites are scarce, existing datasets offer limited coverage, and static rewards fail to adapt as models improve. Recent self-play methods unify code and test generation in a single model, but face a inherent dilemma: white-box access leads to self-collusion where the model produces trivial tests for easy rewards, yet black-box restriction yields generic tests that miss implementation-specific bugs. We introduce Code-A1, an adversarial co-evolution framework that jointly optimizes a Code LLM and a Test LLM with opposing objectives. The Code LLM is rewarded for passing more tests, while the Test LLM is rewarded for exposing more defects. This architectural separation eliminates self-collusion risks and safely enables white-box test generation, where the Test LLM can inspect candidate code to craft targeted adversarial tests. We further introduce a Mistake Book mechanism for experience replay and a composite reward balancing test validity with adversarial difficulty. Experiments on Qwen2.5-Coder models demonstrate that Code-A1 achieves code generation performance matching or exceeding models trained on human-annotated tests, while significantly improving test generation capability.

### 1. 核心问题与整体含义
*   **研究动机**：代码生成的强化学习（RL）高度依赖单元测试提供的可验证奖励。然而，现有人工标注的测试集规模有限且是静态的，无法随模型能力提升而进化。
*   **现有方案缺陷**：
    *   **静态奖励**：测试太简单会导致错误代码过关，太难则导致学习信号消失。
    *   **自博弈（Self-Play）困境**：若模型同时生成代码和测试，在“白盒”模式下会产生**自我勾结**（生成极简测试以刷高奖励）；在“黑盒”模式下则只能生成通用测试，无法发现特定实现的逻辑漏洞。
*   **核心目标**：通过架构分离和对抗学习，构建一个能够动态进化、且能进行针对性“白盒”攻击的自动化代码/测试协同优化框架。

### 2. 方法论
Code-A1 采用代码模型（Code LLM）与测试模型（Test LLM）对抗演化的框架：
*   **核心思想**：将两个模型解耦，赋予相反的目标。Code LLM 目标是“通过更多测试”，Test LLM 目标是“写出能让代码报错的有效测试”。
*   **关键技术细节**：
    *   **对抗性 Rollout**：Code LLM 生成候选代码；Test LLM 在观察候选代码（白盒）的基础上生成针对性测试用例。
    *   **错题本（Mistake Book）机制**：为每个题目维护一个历史失败测试用例库。这起到经验回放的作用，防止 Code LLM 在学习新知识时遗忘如何修复旧漏洞，并为奖励计算提供稳定的基准。
    *   **复合奖励设计**：
        *   **Code LLM 奖励**：基于新生成测试和错题本测试的综合通过率。
        *   **Test LLM 奖励**：平衡**有效性**（测试必须可执行且预测结果正确）与**对抗难度**（新测试应比历史测试更难通过）。
    *   **策略优化**：采用 **GRPO（群组相对策略优化）**。为了平衡算力，引入了 **TopVar（高方差筛选）** 算法，仅选择奖励方差最大的测试组进行更新。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了从 1.5B 到 7B 的三个参数量级，验证了方案的可扩展性。
*   **消融实验**：非常充分，分别验证了**预测答案的必要性**（防止生成不可解测试）、**白盒访问的价值**（证明观察代码能提升攻击性）、以及**错题本的作用**（证明其对训练稳定性的贡献）。
*   **客观性**：通过引入复合指标 **Mul (pass@k × mut@k)** 来平衡测试的有效性与难度，避免了单一指标的偏差。同时与最新的同类研究 CURE 进行了跨模型对比。

### 6. 主要结论与发现
*   **性能超越人工**：Code-A1 训练的代码模型在多个指标上达到或超过了使用人工标注数据训练的模型。
*   **测试能力飞跃**：3B 规模的 Test LLM 经过对抗训练后，其测试生成能力（Mul 得分）超过了 7B 的基础模型，证明对抗演化比单纯堆参数更有效。
*   **协同效应**：在推理阶段，Code-A1 训练的测试模型能更有效地从多个候选代码中筛选出正确答案（Best-of-N）。

### 7. 优点
*   **解决自我勾结**：通过模型解耦，从根本上解决了自博弈模型“作弊”的问题。
*   **动态课程学习**：随着 Code LLM 变强，Test LLM 会自动生成更难的用例，形成天然的难度递增过程。
*   **稳定性高**：错题本机制有效地缓解了对抗训练中常见的灾难性遗忘和训练震荡。

### 8. 不足与局限
*   **依赖 Ground Truth 代码**：在训练阶段仍需要标准答案（Oracle）来校验 Test LLM 生成的测试用例是否正确，这限制了其在完全无标注领域（如前沿科学发现）的应用。

*   **语言单一**：实验主要集中在 Python 语言，尚未验证在其他编程语言或长文本代码库中的泛化能力。
