## 摘要
基于语言模型（LM）的智能体在给定高层动作 API 时表现出色，但在将语言落地为底层控制方面面临挑战。先前的研究让 LLM 为强化学习（RL）生成技能或奖励函数，但这些单次（one-shot）方法缺乏纠正规范错误的反馈。我们提出了 SCALAR，这是一个通过学习到的技能库将 LLM 规划与 RL 耦合的双向框架。LLM 提出带有前提条件和效果的技能；RL 为每个技能训练策略，并反馈执行结果以迭代优化规范，从而提高对初始错误的鲁棒性。关键轨迹分析（Pivotal Trajectory Analysis）通过分析 RL 轨迹来修正 LLM 先验；前沿检查点（Frontier Checkpointing）可选择性地在技能边界保存环境状态，以提高样本效率。在 Craftax 任务中，SCALAR 实现了 88.2% 的钻石收集率，比最佳基准提高了 1.9 倍，并在 9.1% 的时间内到达了侏儒矿山（Gnomish Mines），而先前的方法在这些任务中完全失败。

## Abstract
LM-based agents excel when given high-level action APIs but struggle to ground language into low-level control. Prior work has LLMs generate skills or reward functions for RL, but these one-shot approaches lack feedback to correct specification errors. We introduce SCALAR, a bidirectional framework coupling LLM planning with RL through a learned skill library. The LLM proposes skills with preconditions and effects; RL trains policies for each skill and feeds back execution results to iteratively refine specifications, improving robustness to initial errors. Pivotal Trajectory Analysis corrects LLM priors by analyzing RL trajectories; Frontier Checkpointing optionally saves environment states at skill boundaries to improve sample efficiency. On Craftax, SCALAR achieves 88.2% diamond collection, a 1.9x improvement over the best baseline, and reaches the Gnomish Mines 9.1% of the time where prior methods fail entirely.

#### 1. 核心问题与整体含义
论文针对 **LLM 智能体在底层控制（Grounding）方面的局限性** 展开研究。虽然 LLM 在高层推理和代码生成上表现出色，但在直接控制底层动作时往往失败。现有的“LLM + RL”方法多为“单次（One-shot）”模式：LLM 提供奖励函数或子目标，RL 负责执行。这种模式缺乏反馈机制，一旦 LLM 对环境的前置条件（如资源消耗量）估计错误，RL 训练就会陷入僵局。SCALAR 旨在建立一个**双向闭环框架**，让 LLM 的符号规划与 RL 的底层执行相互修正。

#### 2. 方法论
SCALAR 的核心思想是将技能形式化为符号算子（Operators）和神经选项（Options），并通过反馈循环不断优化。
*   **技能提案（Operator Proposal）**：LLM 根据游戏手册提出技能的符号定义，包括前置条件（Preconditions）、正向效果（Eff+）、负向效果（即资源消耗 Eff-）以及奖励函数。
*   **符号规划与组合**：使用 STRIPS 风格的规划器将已掌握的技能串联，以达到新技能的前置条件状态。
*   **关键轨迹分析（Pivotal Trajectory Analysis）**：这是核心反馈机制。当 RL 成功完成技能后，系统分析成功轨迹，对比实际消耗与 LLM 的初始假设。如果 LLM 预估需要 3 个木材但实际只需 1 个，LLM 会修正算子定义，从而优化后续规划。
*   **前沿检查点（Frontier Checkpointing）**：为了解决长程任务中重复执行前置技能导致的采样效率低下问题，系统会在满足前置条件的状态（“前沿”）保存环境快照，训练时直接从快照恢复。
*   **联合更新**：使用 PPO 算法，在训练目标技能的同时，允许梯度回传至前置技能的策略，确保技能组合的连贯性。

*   **架构限制**：在 Transformer 等有状态架构中，恢复环境状态同时需要恢复网络记忆，实现复杂度较高。
